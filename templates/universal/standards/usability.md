# Usability Standard

Apply when your output is user-facing. Evaluate every change against these criteria.

**Perceivable** — All content must be presentable to users regardless of ability. Provide text alternatives for images. Ensure sufficient colour contrast (4.5:1 minimum). Do not convey information through colour alone.

**Operable** — All functionality must be accessible via keyboard. Focus order must follow logical reading order. Interactive elements must have visible focus indicators. No keyboard traps.

**Understandable** — Labels, instructions, and error messages must be clear and specific. Form errors must identify the field and describe the problem. Navigation must be consistent across pages.

**Robust** — Use semantic HTML elements (`button`, `nav`, `main`, `form`) over generic `div` and `span`. Add ARIA attributes only when semantic HTML is insufficient. Ensure compatibility with assistive technologies.
